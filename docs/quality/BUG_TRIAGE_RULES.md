# Bug Triage Rules

## Priority

- `P0`: blocks ship, risks save data, breaks critical flow, or fails stop-ship criteria
- `P1`: serious quality problem in onboarding, combat readability, creator trust, performance, or accessibility
- `P2`: important but not ship-blocking polish or workflow issue
- `P3`: minor polish or backlog candidate

## Ownership

- Every P0 and P1 issue must have an owner, current status, and next action.
- Save corruption, creator containment, and accessibility blockers must be triaged the same day they are found.
