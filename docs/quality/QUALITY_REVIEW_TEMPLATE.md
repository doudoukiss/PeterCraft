# Quality Review Template

## Pillar checks

- Adventure first
- Systems made visible
- Safe creation
- Low-friction iteration
- World-class craft

## Review questions

- Does this change improve the game promise without harming the learning or maker promise?
- Is the behavior understandable to a child or mentor?
- Is there validation, telemetry, and debug visibility for the change?
- Can a mistake be recovered without damaging progress?
- Is the implementation portable and maintainable?

## Required evidence

- test results
- validation results
- telemetry or debug proof where applicable
- docs or README updates for new contracts
