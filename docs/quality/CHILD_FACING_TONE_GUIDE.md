# Child-Facing Tone Guide

## Rules

- Prefer supportive, clear language over system jargon.
- Explain cause and effect before naming implementation details.
- Validation text should tell the player what to try next.
- Failure language should stay calm and recoverable.
- Debug wording stays in developer/debug panels only.

## Preferred examples

- "That change is safe. Preview it, then apply it when you are ready."
- "That change is not safe yet. Try the suggested range or reset it."
- "Open the Why panel to see the top reasons behind your companion's choice."
